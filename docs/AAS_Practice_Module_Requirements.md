# AAS Practice Module Requirements

## 1. Overview

The AAS Practice Module is the capstone assessment for the Graduate Certificate in Architecting AI Systems. Assessment is based on a group project.

## 2. Prerequisites

Participants should complete or be exempted from all required course modules before the practice project.

## 3. Team and Effort

- Team size: 4 to 5 members
- Estimated effort: about 15 person-days per member
- Work is distributed across multiple weeks

## 4. Proposal Requirements

The project proposal should clearly cover:

- Motivation and problem statement
- System overview and architecture
- Expected outcomes and deliverables
- Team roles and responsibilities
- Technology stack
- Risk assessment and mitigation
- Timeline and milestones
- Effort estimation

## 5. Final Deliverables

- Source code with clear structure
- Technical documentation
- Test evidence
- Final report and presentation/demo materials

## 6. Assessment Focus

- Technical quality and completeness
- Responsible AI and security considerations
- Collaboration and project management process
- Clarity of documentation and presentation

## 7. Recommended Working Practices

- Start with a realistic scope
- Maintain clear ownership and communication
- Use version control and CI from the beginning
- Test continuously, not only at the end
- Keep documentation updated during implementation

## 8. Common Risks

- Over-scoped plan
- Late integration across team modules
- Weak test coverage
- Incomplete documentation

## 9. Practical Submission Checklist

- Proposal finalized
- Code and docs organized
- Tests executed with results recorded
- Final report completed
- Demo flow rehearsed
